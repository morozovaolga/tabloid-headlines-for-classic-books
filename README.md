# 🎯 Генератор жёлтых заголовков с RAG

Автоматическая генерация скандальных заголовков для художественной литературы с использованием RAG (Retrieval-Augmented Generation).

## 🚀 Что это?

Скрипт берёт случайные книги из датасета HuggingFace, находит похожие примеры из базы знаний и генерирует качественные жёлтые заголовки через локальную LLM.

## ✨ Ключевые фичи

- ✅ **RAG система** — автоматический поиск похожих примеров
- ✅ **База из 10 качественных примеров** — классика мировой литературы
- ✅ **Автоматическое пополнение базы** — лучшие заголовки добавляются сами
- ✅ **Оценка качества** — скоринг каждого заголовка
- ✅ **Выбор лучшего из 5** — генерируем несколько вариантов
- ✅ **Датасет HuggingFace** — 1271 книга

## 🔧 Как это работает

### 1. RAG находит похожие примеры

```
Новая книга: "Идиот"
Сюжет: "Князь влюбляется в красавицу... Купец из ревности..."

       ↓ TF-IDF поиск

Похожие примеры:
1. "Купец убил красавицу из ревности" (похожесть: 78%)
2. "Студент убил старуху" (похожесть: 45%)
3. "Офицер похитил княжну" (похожесть: 32%)
```

### 2. Показывает примеры модели в промпте

```
ПРИМЕРЫ ОТЛИЧНЫХ ЗАГОЛОВКОВ:
1. "Студент заплатил страшную цену за свои убеждения"
2. "Офицер похитил княжну: скандал потряс общество"
3. "Встреча с незнакомкой обернулась трагедией для инвалида"

Создай заголовок для: [новая книга]
```

### 3. Модель генерирует в том же стиле

```
→ "Купец зарезал возлюбленную из ревности: князь сошёл с ума"
   (оценка: 0.89)
```

### 4. Автоматически добавляет лучшие в базу

Если оценка ≥ 0.85 → новый пример сохраняется в базу RAG!

## 📁 Выходные файлы

После запуска создаются:

### yellow_headlines_rag.jsonl
```json
{"input": "Название: Война и мир\n...", "suggestions": ["Офицер..."], "meta": {...}}
```

### yellow_headlines_rag.json
То же самое в читаемом формате.

### examples_db.json
База RAG примеров (автоматически пополняется).

### yellow_headlines_rag_skipped.txt
Список пропущенных книг с причинами.

## ⚙️ Настройки

В файле `generate_headlines_final.py`:

```python
# Модель
OLLAMA_MODEL = "qwen2.5:3b"

# Сколько книг обработать
BOOKS_PER_RUN = 10

# Сколько попыток генерации
num_attempts = 5  # в функции generate_with_rag

# Порог для автодобавления в базу RAG
if score >= 0.85:  # Можно изменить на 0.80 или 0.90
    rag.add_example(...)
```

## 🎓 Как улучшить качество

### 1. Используй лучшую модель

```bash
# Вместо qwen2.5:3b попробуй:
ollama pull mistral:7b      # +10% качества
ollama pull qwen2.5:14b     # +15% качества (медленнее)
```

### 2. Добавь свои примеры в базу

Отредактируй `examples_db.json`:

```json
{
  "examples": [
    {
      "book": "Твоя книга",
      "author": "Автор",
      "plot_keywords": ["ключевые", "слова", "из", "сюжета"],
      "headline": "Твой идеальный заголовок",
      "score": 1.0
    }
  ]
}
```

### 3. Увеличь количество попыток

В коде измени:
```python
generate_with_rag(..., num_attempts=10)  # Было 5
```

### 4. Настрой промпт

В функции `build_rag_prompt()` можно:
- Добавить больше правил
- Изменить стиль инструкций
- Добавить negative examples
