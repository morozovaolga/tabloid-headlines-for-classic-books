# Генератор «жёлтых» заголовков — веб‑приложение (v2)

Лёгкое офлайн‑веб‑приложение для генерации 5 кликбейт‑заголовков по сюжету книги и быстрой разметки датасета для обучения.

- Источник сюжетов: локальный датасет Hugging Face `slon-hk/BooksSummarizationRU` (кэш в `.cache/hf_datasets`).
- Генерация: Ollama (по умолчанию `qwen2.5:3b`).
- 5 кандидатов за раз + пакетная разметка good/bad с правками текста — всё пишется в `labels.jsonl`.
- Нормализация регистра: только имена собственные и аббревиатуры с заглавной; всё прочее — строчными (исправляет «Title Case»).
- Быстро: датасет загружается один раз на процесс, дальше выборки мгновенные.
- Внешние API (Svet, Яндекс, Литрес) удалены — ничего лишнего и тормозного.

## Установка
- Python 3.10+
- Зависимости: `pip install -r requirements.txt`
- Ollama: `ollama serve` и модель `ollama pull qwen2.5:3b`

Опционально для офлайна/скорости:
- `HF_DATASETS_CACHE=.cache/hf_datasets` (уже по умолчанию)
- `HF_HUB_OFFLINE=1` — запрет сетевых обращений к Hub

## Запуск
```
uvicorn app:app --reload --port 8000
```

## Маршруты
- `/` — главная
- `/simple` — ручной ввод: название/автор/аннотация → 5 заголовков → разметка
- `/discover` — 10 случайных книг из датасета (заголовок + автор + сюжет) → 5 заголовков на книгу → пакетная разметка
- `/random` — показать случайные примеры из локальных *.jsonl (если есть)
- `/labels` — просмотр собранного датасета разметки
- `/healthz` — проверка готовности

## Разметка (fine‑tune)
Все метки пишутся построчно в `labels.jsonl` c полями: `title`, `author`, `original`, `edited`, `label`, `plot`, `model`, `gen_source`, `ts` и др.

Собрать обучающий набор (инструкционный формат, только «good»):
```
python prepare_finetune.py --labels labels.jsonl --out train_instruct.jsonl --format instruct --only good
```

## Обучение модели (LoRA)
- Рекомендуемый стартовый объём: 200–500 «good» примеров (лучше с правками).
- Запустите дообучение на компактной чат‑модели (подходит даже CPU/GPU 8–12GB):
```
python train_lora.py \
  --data train_instruct.jsonl \
  --fmt instruct \
  --model TinyLlama/TinyLlama-1.1B-Chat-v1.0 \
  --epochs 1 \
  --batch 2 \
  --grad-accum 8 \
  --out outputs/lora-headlines
```
- На выходе появится папка адаптера LoRA: `outputs/lora-headlines/`.

### Подключение адаптера в веб‑приложении
На странице `/simple` включите «HF/LoRA вместо Ollama», укажите:
- База: `TinyLlama/TinyLlama-1.1B-Chat-v1.0`
- Путь к адаптеру: `outputs/lora-headlines`
Затем генерируйте — веб‑интерфейс использует ваш LoRA.

Под капотом
- `prepare_finetune.py` собирает пары «инструкция→ответ» из `labels.jsonl` (или формат simple).
- `train_lora.py` обучает LoRA‑слой поверх базовой модели и сохраняет адаптер.
- Для инференса HF/LoRA есть `hf_infer.py`, а в UI — чекбокс «Использовать HF/LoRA».

## О генерации
- Модель Ollama настраивается через `HEADLINE_MODEL` (или в UI по умолчанию). По умолчанию: `qwen2.5:3b`.
- На книгу генерируется 5 кандидатов; «лучший» определяется простым скорером (длина, «драматизм», релевантность), но размечать лучше все 5 — это полезнее для обучения.
- Капитализация: первая буква предложения + имена собственные/аббревиатуры. Остальные слова — строчные.

## Производительность
- Первый вызов /discover инициализирует датасет (несколько секунд). Далее всё мгновенно благодаря кэшу в памяти.
- Для офлайна можно выставить `HF_HUB_OFFLINE=1`.

## Полезное
- Резервные источники сюжетов: Wikipedia остаётся доступной внутри генератора (`generate_headlines_v2.py`) как fallback.
- Для локального теста вне веба можно использовать CLI‑скрипт `generate_headlines_v2.py` (опционально).

## Лицензия и дисклеймер
Инструмент предназначен для работы с художественными произведениями и научно‑исследовательских целей. Не генерирует материалы о реальных людях/событиях.

## Быстрый старт: запуск сервера

- Активируйте виртуальное окружение (если используется).
- Запустите приложение командой:

```
uvicorn app:app --reload --port 8000
```

После запуска откройте `http://127.0.0.1:8000/` в браузере.
## Как работает генерация

- Источники сюжета:
  - Ручной ввод: аннотация + первые 1000 знаков текста.
  - Wikipedia: раздел «Сюжет/Краткое содержание/…» (если включено) — лишние блоки «Примечания/Ссылки/…» удаляются.
  - Объединение: функция `combine_plot` склеивает всё, чистит сноски `[...]`, режет до ~2000 символов.

- Промпт (build_prompt):
  - Стиль: `yellow` (кликбейт) или `neutral` (умеренно‑медийный).
  - Тумблеры: `SENSITIVE_MODE`, `ABSURD_MODE`, `HEADLINE_HUMOR`.
  - WWII‑исключение: если в сюжете детектируется контекст Второй мировой войны (ключевые слова + годы 1941–1945), абсурд/юмор приглушаются, добавляется просьба формулировать уважительно.
  - Анти‑мета правило: «Не начинай фразами “Спасибо за предоставленный текст”, “Краткое…”, “В данном тексте…”.»

- Очистка ответа (post‑process):
  - `clean_generated_headline` убирает реальные и «литеральные» переносы (`\n`, `\`n), кавычки, точку в конце, сжимает пробелы, отрезает слишком длинные заголовки, приводит к предложному регистру.
  - `normalize_proper_names` аккуратно капитализирует имена/фамилии и инициалы, не задирая обычные слова (союзы/предлоги остаются строчными).

- Валидация (жёсткий отсев):
  - Длина: 8–15 слов.
  - Запрещены кавычки, скобки, угловые/квадратные скобки, служебные вставки.
  - Только художественная проза: заголовки вида «автор/написал/писатель…» отбрасываются.
  - Анти‑мета фильтр: ответы вида «спасибо за предоставленный текст… / по данному тексту… / вот краткое… / данный текст…» отбрасываются.

- Скоринг (объяснимый):
  - База по длине: `s = 0.5 - 0.03 * |11 - N|`, где `N` — число слов (8–15 даёт лучший диапазон). Ограничение результата: 0..1.
  - «Релевантность»: +0.04 за каждое пересечение лемм заголовка и сюжета (до +0.15).
  - Юмор/приёмы (если `HEADLINE_HUMOR=1`):
    - Детектор фиксирует: противопоставление, парадокс, гиперболу, далёкое сравнение, стилистический контраст, намёк/двусмысленность, повтор.
    - Бонус: 0.05 за первый приём +0.03 за следующий, максимум +0.18. В WWII‑контексте бонус умножается на 0.4.
  - Мета‑ответ ("спасибо за…", и т. п.): жёсткий штраф → score = 0.0.

- «Reasons» в UI (подсказки):
  - `length=<N>` — длина в словах.
  - `relevance_overlap=<K>` — число общих лемм с сюжетом.
  - `humor=<…>` — найденные приёмы (если включено).
  - `ww2_context` — обнаружен контекст WWII.
  - `meta-like` — сработал анти‑мета фильтр (кандидат будет отсеян).

- Выбор top‑k и сохранение:
  - Генерируется ≥5 кандидатов (модель Ollama по умолчанию `qwen2.5:3b`).
  - Валидные кандидаты сортируются по score, сохраняются top‑5.
  - На /labels сохраняются обе метки: `user_label` (ваша) и `model_label` (по порогу), плюс `model_score`.

- Порог модели для «good»:
  - `HEADLINE_GOOD_TH` (по умолчанию 0.65). Можно изменить через переменную окружения.

### Переключатели окружения
- `HEADLINE_MODEL` — модель Ollama (по умолчанию `qwen2.5:3b`).
- `HEADLINE_ATTEMPTS` — число попыток генерации.
- `HEADLINE_LANG` — язык Wikipedia/интерфейса (`ru`).
- `HEADLINE_ABSURD` — допуск абсурда (true/false), автоматически глушится при WWII.
- `HEADLINE_HUMOR` — учёт «шутливых» приёмов (true/false).
- `HEADLINE_OUT` — JSONL c веб‑генерацией.
- `HEADLINE_LABELS` — JSONL с разметкой.
- `HEADLINE_GOOD_TH` — порог «good» для `model_label`.
- `HEADLINE_USE_HF` / `HEADLINE_HF_MODEL` / `HEADLINE_LORA_ADAPTER` — опции локального HF‑инференса.
